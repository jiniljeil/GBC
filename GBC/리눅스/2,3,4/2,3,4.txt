2-1. 리눅스 파일의 종류와 특징
    파일
    -일반 파일 : 데이터를 저장하는데 주로 사용 (ex : 각종 텍스트 파일, 실행 파일, 이미지 파일)
       바이너리 파일 : 데이터가 바이너리 형태로 저장 (ex : 실행 파일, 이미지 파일)
      
     -디렉터리 : 파일로 취급 , 디렉터리 = 폴더

     -심벌릭 링크 : 원본 파일을 대신하여 다른 이름으로 파일명을 지정한 것, (윈도우 바로가기 파일과 비슷하다고 볼 수 있음)

     -장치파일 : 리눅스시스템에 부착된 장치들을 관리하기 위한 특수 파일. (하드디스크나 기보드 같은 각종 장치도 파일로 취급)

2-2. 디렉터리 계층 구조 (사진 첨부)
     수많은 파일을 관리하기 위해 디렉터리를 사용하며 파일을 효율적으로 관리하기 위해 디렉터리를 계층적으로 구성 = 트리구조 
      
      상위(부모) 디렉터리 : 자신을 포함하고 있는 상단의 디렉터리 (루트 디렉터리는 유일하게 상위 디렉터리가 없음)
      하위(서브) 디렉터리 : 디렉터리의 아래에 있는 디렉터리
      
      홈디렉터리 : 자신의 홈 디렉터리 아래에 파일이나 서브 디렉터리를 생성하며 작업 가능 (~ 사용자 이름)
 
    -절대 경로명 : 항상 루트 (/)디렉터리부터 시작해서 특정 파일이나 디렉터리의 위치까지 이동하면서 거치게되는 모든 중간 디렉터리의 이름을 표시
    -상대 경로명 : 현재 디렉터리를 기준으로 시작해서 서브 디렉터리의 이름을 추가해서 표시

2-3. 디렉터리 이동 및 확인 
    현재 디렉터리 확인하는 명령 : pwd (print working directory)
    디렉터리 이동하기 명령 : cd (change directory) [디렉터리명] , 홈 디렉터리로 이동 : cd ~ , cd /home/사용자 이름 등등
    디렉터리 내용 출력 명령 : ls [옵션] [파일 또는 디렉터리명]
     옵션 -a : 숨김 파일을 포함하여 모든 파일 목록을 출력
             -d : 지정한 디렉터리 자체의 정보를 출력
              -l : 파일의 상세 정보를 출력
             -R : 하위 디렉터리 목록까지 출력
             -F : 파일을 구분하는 기호가 표시된 상태로 출력 
               등등이 존재

     파일 이름 뒤에 /이 붙으면 디렉터리 @는 심벌릭 링크 *는 실행 파일 아무 표시도 없으면 일반 파일       
     ls 에대한 심벌릭 링크 : dir
     ls -l 에 대한 심벌릭 링크 : vdir

2-4. 디렉터리 생성 및 삭제
    디렉터리 생성하는 명령 : mkdir [옵션] 디렉터리명
    옵션 -p : 중간 디렉터리 자동으로 만들기 , 만들고 난 후 ls -R 디렉터리명 (명령어)을 입력하여 확인
     
    디렉터리 삭제하는 명령 : rmdir 디렉터리명 , 단 해당 경로에 가서 삭제 가능
       
2-5. 다양한 명령을 통한 파일의 내용 확인
    텍스트 파일의 내용을 출력하는 명령 : cat [옵션] 파일명 
     옵션 -n : 행 번호를 붙여서 출력 

     화면 단위로 파일 내용을 출력해주는 명령 : more [옵션] 파일명
     옵션 +행 번호 : 출력을 시작할 행 번호를 지정.
     more 명령을 종료하는 키 : q 키 
     
     파일 내용을 앞뒤로 스크롤하며 이동할 수 있게 하는 명령어 : less 파일명
     파일 뒷부분 출력하는 명령어 : tail [옵션]  파일명 
     옵션  +행 번호 : 지정한 행부터 끝까지 출력함.
             -숫자 : 화면에 출력할 행의 수를 지정 (기본 값은 10)
             -f : 파일 출력이 종료되지 않고 주기적으로 계속 출력

2-6. 파일 복사, 이동, 삭제
    파일 복사 명령어 : cp (copy) [옵션] 파일명1/디렉터리명1  파일명2/디렉터리명2
    옵션 -i : 대화식 복사 방법으로 파일명2가 이미 존재할 경우 덮어쓸 것인지 물어본다.
           -r : 디렉터리를 복사할 때 지정한다.
    ex ) cp file1 file2

    파일 이동 명령어 : mv (move) [옵션] 파일명1/디렉터리명1 파일명2/디렉터리명2
    옵션 -i : 파일명2/디렉터리명2가 존재하면 덮어쓸 것인지 물어본다.
    ex ) mv file1 file2

    파일 삭제 명령어 : rm (remove) [옵션] 파일명/디렉터리명 
    옵션 -i : 지정한 파일을 삭제할 것인지 대화식으로 확인한다.
           -r : 지정한 디렉터리를 삭제한다. 
    ex) rm file , rm -r dir

2-7. 파일 링크의 특징, 하드 링크와 심벌릭 링크  (부족)
    파일 링크 생성 명령어 :  ln  [옵션] 원본파일명 링크파일명
    옵션 -s : 삼벌릭 링크 파일을 생성함.
     ex) ln -s data1 data1.sl 

    심벌릭 링크와 하드 링크의 차이 
    1. 파일의 종류가 l(소문자 L)로 표시된다.
    2. 하드 링크의 개수가 하나이다. 즉, 원본 파일에 이름을 추가하는 것이 아니다.
    3. 파일 이름 뒤에 원본 파일의 이름이 표시된다.(->data1).
    4. inode 번호가 원본 파일과 다르다. 즉, 원본 파일과 심벌릭 링크 파일은 별개의 파일임. 
    
    심벌릭 링크의 특징 : 심벌릭 링크의 내용을 출력하면 원본 파일의 내용이 출력됨.
     
2-8. 파일 내용과 위치 검색
    빈 파일 만들기, 수정 시간 변경하기 명령어 : touch [옵션] 파일
    옵션 -a : 접근 시간만 변경한다.
           -m : 수정 시간만 변경한다.
           - t : 시간을 직접 입력한다.
    현재 시각을 확인하는 명령어 : date 
    
    파일 내용 검색하기 명령어 : grep [옵션] 패턴 [파일명]
    옵션 -i : 대,소문자를 모두 검색    
            -l : 해당 패턴을 포함하는 파일 이름을 출력함.
           -n : 행 번호를 출력
    ex ) grep root /etc/passwd
           grep -n unix ~/",txt
           grep -l hello ".c

   파일 찾기 명령어 : find 경로 검색 조건 [동작]
   옵션 -name 파일이름 : 파일 이름으로 검색
           -type 파일 종류 : 파일의 종류로 검색
           -user loginID : 지정한 사용자가 소유한 모든 파일을 검색
           -perm 접근 전환 : 지정한 사용 권한과 일치하는 파일을 검색
   동작 -exec 명령 {} \; : 검색된 파일에 명령을 실행
          -ok 명령 {} \; : 사용자의 확인을 받아서 명령을 실행
          -print : 검색된 파일의 절대 경로명을 화면에 출력
          -ls : 검색 결과를 긴 목록 형식으로 출력 
    ex ) find ~ -name hello.c
          find /tmp -user user10 -exec rm {}\;

   명령의 위치 찾기 명령어 : whereis [옵션] 명령 , which
   옵션 -b : 바이너리 파일만 검색
           -m : 메뉴얼 파일만 검색
           -s : 소스 파일만 검색   

3. 리눅스의 문서 편집기
   편집기 종류 
   -행 단위 편집기 : ed, ex, sed
   -화면 단위 편집기 : vi,emacs(이맥스)
   -GUI 편집기 : gedit
    
   모드형 편집기 (대표적인 예 : vi 편집기 )
   입력 모드와 명령 모드가 구분
    
   비모드형 편집기 
   두 모드가 구분X
   
   입력 모드 : 텍스트를 입력할 수 있는 모드 
   명령 모드 : 텍스트를 수정하거나 삭제하고 복사와 붙이기 등 편집을 하는 모드 

   vi 사용법 (사진첨부)
   vi [파일명] : 지정한 파일을 편집하고 , 파일명을 지정하지 않으면 빈 파일이 열리고, 이 빈 파일의 파일명은 별도로 지정 
   
   파일 저장과 종료 명령키 
  마지막 행 모드 
             :q vi에서 작업한 것이 없을 때 그냥 종료
             :q! 작업한 내용을 저장하지 않고 종료
             :w 파일명 작업한 내용을 저장 만한다.
             :wq,wq! 작업한 내용을 저장하고 vi 종료
   명령모드 -ZZ : 작업한 내용을 저장하고 vi 종료

   명령 키를 사용해 입력모드로 전환 
    i : 커서 앞 입력
   a : 커서 뒤 입력
   
   내용 수정 명령 키 
   r  : 한 글자 수정하기
   cw, #cw : 커서 위치부터 현재 단어의 끝까지 수정
   s, #s : 커서 위치부터 Esc 키를 입력할 때 까지 수정
   cc : 커서가 위치한 행의 내용을 모두 수정
   C : 커서 위치부터 행의 끝까지 수정
   
   내용 삭제 명령 키 
   x, #x : 커서 위치의 (한) 글자를 삭제, #에는 삭제할 글자 수를 지정 ex) #3x
   dw, #dw : 커서 위의 단어를 삭제, # 위와 삭제할 단어 수 지정
   dd, #dd : 커서 위치의 행을 삭제 # 위에 삭제할 행의 수 지정
   D : 커서 위치부터 행의 끝까지 삭제 
  
  명령 취소하는 명령키 
  u : 명령을 취소
  U : 해당 행에서 한 모든 명령을 취소
  :e! : 마지막으로 저장한 내용 이후의 것을 버리고 새로운 작업

  복사하기 또는 잘라서 붙이기
  p : 복사하여 붙이기 명령키
  dd : 행을 삭제하는 것과 동시에 잘라내기 명령키
  yy : 해당 행이 복사되는 명령키
  ex) 1G --> -->yy --> jj --> p  : 1번 째 행을 복사해서 커서의 위치에 붙여넣기
  
  검색하기 명령키
  /문자열 : 문자열을 아래 방향으로 검색
  ?문자열 : 문자열을 위 방향으로 검색
  n : 원래 찾던 방향으로 다음 문자열을 찾음.
  N : 역방향으로 다음 문자열을 찾음.

 vi는 사용자가 환경을 설정할 수 있또록 set 명령을 제공
 vi의 환경을 설정하는 데 세 가지 방법 사용
1. 사용자 홈 디렉터리에 .exrc 파일로 저장
2. 환경 변수 EXTINIT에 지정.
3. vi의 마지막 행 모드에서 명령으로 실행 가능

vi 환경 설정 명령 키 
set nu : 파일 내용의 각 행에 행 번호를 표시 
set nonu : 행 번호를 감춘다.
set list : 눈에 보이지 않는 특수 문자를 표시
set nolist : 특수 문자를 감춤
set showmode : 현재 모드 표시 
set noshowmode : 현재 모드를 감춘다.
set : set로 설정한 모든 vi 환경 설정 값을 출력
set all : 모든 vi 환경변수와 현재 값을 출력

4. 셸 사용하기 
   셸의 기능 : 명령어 해석기 기능 , 프로개르밍 기능, 사용자 환경 설정 기능 등 크게 세가지로 구분
   셸의 종류 : 본 셸, C 셸 , 콘 셸, 배시 셸  
    에일리어스 설정이나 초기화 파일, 스크립트를 작성하는 문법 등에 약간씩 차이를 가짐.

   셸의 기본 사용법
   자신의 셸 확인 :  grep 사용자 이름 /etc/passwd 
   기본 셸 바꾸기 : chsh [옵션] [사용자명]
   옵션 -s shell : 지정하는 셸(절대 경로)로 로그인 셸을 바꾼다.
           -l :  /etc/shells 파일에 지정된 셸을 출력
   ex ) chsh -l, chsh
   
   현재 바꿀 수 있는 셸 종류 출력 : cat /etc/shells 
   
   로그인 셸과 서브셸 (사진 첨부)

   셸 내장 명령
   대표적인 셸 내장 명령 : cd 
   file /usr/bin/cd  : file 명령은 파일의 종류를 알려줌.

   출력하기 명령어 
    echo [-n] [문자열 ..]
   옵션  -n : 마지막에 줄 바꿈하지 않는다.

    printf 형식 [인수]
   옵션 %d, \n 등 C 언어의 printf 함수 형식을 지정 
   
   특수 문자 사용하기 
   특수 문자 : *, ? , | , ; . [] . ~. " ", ' '. `` 등이 존재
   * : 주로 여러 파일의 이름을 나열할 때 파일 이름을 간단히 표시하는데 사용, 명령을 실행할 때 파일 이름을 적어야하는 자리에 *를 사용하면 모든 파일을 나타냄
   
   ?,[] : 모두 하나의 문자를 나타내는데 사용, ?는 길이가 1인 임의의 한 문자를, []는 괄호 안에 포함된 문자 중 하나를 나타내며 사용할 여러 문자를 나열 및 범위 지정 
   
    ~,- : 디렉터리를 나타내는 특수 문자 .
    ~만 사용하면 현재 작업 중인 사용자의 홈 디렉터리를 나타냄 .
     다른 사용자의 로그인 ID와 함께 사용하면 해당 사용자의 홈 디렉터리를 나타냄.  
     -는 cd 명령으로 디렉터리를 이전하기 직전의 작업 디렉터리를 나타냄.
      ex ) cd - : 이전 작업 디렉터리로 이동 

   ;(세미콜론),|(파이프) : 명령과 명령을 연결 , ; 는 연결된 명령을 왼쪽부터 차례로 실행 , |는 왼쪽 명령의 실행 결과를 오른쪽 명령의 입력으로 전달
   ' ' ," " : 문자를 감싸서 문자열로 만들어주고, 문자열 안에 사용된 특수 문자의 기능을 없앰.
     ' ' 는 모든 특수 문자를, " " 는 %,`,\을 제외한 모든 특수 문자를 일반 문자로 간주
 
    `` : 셸은 ` ` 로 감싸인 문자열을 명령으로 해석하여 명령의 실행 결과로 바뀌게 됨.
    
    \ 는 특수 문자 바로 앞에 사용되는데 해당 특수 문자의 효과를 없애고 일만 문자처럼 처리한다.
   
    >,< , >> 는 입출력의 방향을 바꾸는 특수 문자 

   입출력 방향 바꾸기
   표준 입출력 장치를 파일로 바꾸는 것을 '리다이렉션' 이라함.
   
   리다이렉션 두가지 방법
   1. 파일 덮어쓰기 : >  명령  > 파일 이름  
   2. 파일에 내용 추가하기 : >>  명령 >> 파일이름

   오류 리다이렉션 
   기능 : 표준 오류 메세지를 파일에 저장
   명령어 : 명령 2> 파일 이름 
  
   표준 출력과 표준 오류를 한 파일로 리다이렉션 하는 방법
   1. 명령의 정상 실행 결과를 파일로 리다이렉션한다
   2. 그 명령 전체의 오류 메세지를 1번 파일로 리다이렉션한다.

   입력 리다이렉션 명령어 : < , 명령 < 파일이름
    
   배시 셸 환경 설정 
   셸은 셸의 환경을 설정하기 위한 값을 저장 할 수 있도록 셸 변수와 환경 변수를 제공
   셸 변수 : 해당 셸의 지역 변수
   환경 변수 : 현재 셸과 서브 셸에서 공통적으로 사용할 수 있는 전역 변수 
   
   셸 변수 출력 명령어 : set, env 
   set : 셸 변수와 환경 변수를 모두 출력
   env : 환경 변수만 출력 

   주요 셸 환경 변수 
   HISTSIZE : 히스토리 저장 크기 
   HOME : 사용자 홈 디렉터리의 절대 경로
   LANG : 사용하는 언어 
   LOGNAME : 사용자 계정 이름
   PATH : 명령을 탐색할 경로 
   PWD : 작업 디렉터리 절대 경로 
   SHELL : 로그인 셸 
   
   특정 변수 출력하기 : echo $환경변수 
   
   셸 변수 설정 :  변수 이름 = 문자열   ex ) SOME = test 
   환경 변수 설정 : export [-n] [셸 변수] 
   옵션 -n  : 환경 변수를 셸 변수로 바꿈. 
   기능 : 지정한 셸 변수를 환경 변수로 바꿈 .
   변수 해제하기 
   기능 : 지정한 변수를 해체 , 명령어 : unset 
   
   에일리어스 와 히스토리 
   에일리어스  = 별명 
   기능 : 기존의 명령을 대신하여 다른 이름을 붙일 수 있도록 하는 기능 
   형식 : alias 이름 = '명령' 
   
   에일리어스 해체 명령어 : unalias 지정한 명령어
   
   히스토리 
   기능 : 명령어를 입력하는 경우 오타로 인해 그 명령어를 다시 입력해야하는 경우 이전에 입력한 명령을 다시 불러 사용하는 기능 (히스토리를 출력)
   명령어 : history
  
   명령 재실행하기 : !
   !! : 바로 직전에 수행한 명령을 재실행
   !번호 : 히스토리에서 해당 번호의 명령을 재실행
   !문자열 : 히스토리에서 해당 문자열로 시작하는 마지막 명령을 재실행
   *잘못 입력한 명령어 수정하는 경우 : 프롬프트에서 윗쪽 화살표 누르면 직전에 입력했던 명령어가 다시 나타남.

   히스토리 저장하기: more ~/.bash_history

   프롬프트 설정
   환경 변수 :PS1에 프롬프트로 사용할 문자열을 저장
   이스케이프 문자 : \으로 시작하는 특별한 문자 
   이스케이프 문자는 화면에 문자 그대로 출력되지 않고 셸이 문자의 의미를 해석하여 실행함.
   ex) \u : 사용자 이름
         \h : 호스트 이름
         \W : 현재 작업 디렉터리 이름 
         \s : 셸 이름 
         \! : 현재 명령의 히스토리 번호 
         \[ : 출력하지 않을 문자열의 시작 부분 
        \a : ASCLL 종소리 문자  등등
        ex) PS1='`uname -n` \! $'
              PS1='[\u \T] \!$ ' 
    
   컬러 프롬프트 설정
   형식 : PS1= '\[\e[x;y;nm\] 프롬프트 \[\e[x;y;0m\]'
    \[ 는 화면에 출력하지 않을 문자의 시작
   \e 터미널 제어
   x;y 컬러 번호
     n 밑줄이나 역상 표시 등 특수 기능

   프롬프트 컬러 번호 (사진 첨부)
   프롬프트 특수 기능 번호 (사진 첨부)
   
   환경 설정 파일 
   사용자가 로그인할 때마다 자동으로 실행되는 명령을 저장한 것
   ->로그인할 때마다 에일리어스 등을 새로 설정하는 번거로움을 피하는 방법에 유용
   /etc/profile ,  /etc/bashrc/ , /etc/profile.d/*.sh

   사용자 환경 설정 파일
   각 사용자의 홈 디렉터리에 숨김 파일로 있으며 사용자가 내용을 수정하고 관리할 수 있음.
   ~/.bash_profile , ~/.bashrc , ~/.bash_logout
